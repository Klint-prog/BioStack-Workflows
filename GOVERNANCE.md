# Governança

## Modelo atual

BioStack Workflows começa com governança individual. O mantenedor inicial é responsável por:

- Definir o escopo das fases.
- Revisar mudanças antes de avançar.
- Manter o projeto alinhado ao MVP.
- Evitar expansão prematura de escopo.
- Registrar decisões relevantes em `docs/audit-log.md`.

## Abertura futura

À medida que o projeto amadurecer, a governança poderá ser aberta para mantenedores adicionais. Critérios sugeridos:

- Contribuições técnicas consistentes.
- Respeito às decisões de arquitetura.
- Compromisso com reprodutibilidade e auditoria.
- Capacidade de revisar documentação, testes e workflows.

## Decisões técnicas

Decisões relevantes devem ser registradas no audit log, incluindo:

- Mudanças de escopo.
- Adoção ou remoção de dependências.
- Alterações de arquitetura.
- Correções de inconsistências entre fases.
